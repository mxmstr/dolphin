### High-Level Summary

This VR fork of Dolphin 5.0 integrates stereoscopic 3D rendering and 6-DOF (six degrees of freedom) head tracking for VR HMDs like the Oculus Rift and HTC Vive. The implementation is deeply integrated into the video backend, modifying both the Direct3D and OpenGL renderers.

The core of the implementation involves intercepting the game's rendering pipeline to:
1.  **Inject VR Camera Control:** It replaces the game's standard camera projection with a stereoscopic one that incorporates the player's head position and orientation from the VR headset.
2.  **Render for Each Eye:** The main render loop is modified to draw the entire scene twice—once for the left eye and once for the right—each with a slightly different viewpoint to create the 3D effect.
3.  **Handle UI and HUD:** A sophisticated system analyzes rendering commands to differentiate between the 3D world, the skybox, and 2D UI elements (the HUD). It then correctly positions these elements in 3D space, making the HUD feel like a pane of glass in front ofthe player.
4.  **Implement Custom Timewarp:** A custom "Synchronous Timewarp" or "Opcode Replay" feature was developed. This system records the graphics commands for a single game frame and can "replay" them multiple times with updated head-tracking information to generate intermediate frames. This smooths out gameplay and reduces judder when a game's framerate is lower than the HMD's refresh rate (e.g., making a 30fps game feel like 75/90fps in VR).

The fork also includes numerous game-specific hacks, extensive configuration options, and developer tools like an "AR Brute Forcer" to help find memory patches that disable in-game culling, which is often necessary for a comfortable VR experience.

### Key Implementation Details

*   **VR SDK Integration:** The core VR logic is abstracted in `VideoCommon/VR.cpp`. Platform-specific implementations for Direct3D and OpenGL are in `VideoBackends/D3D/VRD3D.cpp` and `VideoBackends/OGL/VROGL.cpp` respectively. These files handle communication with the VR runtimes (like Oculus SDK and OpenVR), including initialization, head tracking, and frame submission.

*   **Stereoscopic Rendering:** The main rendering function, `Renderer::SwapImpl`, was overhauled. It now contains a dedicated VR path that calls `VR_BeginFrame`, renders the scene for the left eye, then renders it again for the right eye, and finally calls `VR_PresentHMDFrame` to send the images to the headset. It uses a Geometry Shader for efficient single-pass stereo rendering where supported.

*   **Camera and World Transformation:** `VertexShaderManager` and the new `VRGameMatrices.cpp` are the heart of the VR camera system. They analyze the game's projection and modelview matrices to understand the scene. They then apply a series of transformations: scaling the world to a 1:1 "lifesize" scale, stabilizing the game camera to reduce motion sickness, applying the player's head tracking, and finally applying the stereo projection for each eye.

*   **UI and HUD Handling:** The system is designed to intelligently separate 2D HUD elements from the 3D world. It uses heuristics based on the projection type (orthographic vs. perspective) and viewport dimensions to classify rendering commands. 2D elements are then rendered onto a virtual "pane of glass" at a configurable distance from the player, making them usable in VR.

*   **Synchronous Timewarp / Opcode Replay:** This is a custom solution to combat low-framerate judder. `OpcodeDecoding.cpp` was modified to record all graphics commands for a frame into a buffer. `Render.cpp` then triggers a "replay" of these commands multiple times between actual game frames, each time with updated head tracking data. This creates intermediate, timewarped frames that make head movement feel smooth even if the game itself is running at a lower FPS.

*   **Developer/Hacking Tools:** The "AR Brute Forcer" (`ARBruteForcer.cpp`) is a powerful tool added to help create VR profiles for games. It systematically patches game functions with a `return` instruction and takes a screenshot to see the effect. This is used to find functions responsible for object culling, which can then be disabled with a patch to prevent objects from disappearing at the edge of the player's view in VR.

### File-by-File Breakdown

Here is a list of all modified and new files from the diffs and a description of the changes.

#### New VR-Centric Files

*   **`Source/Core/VideoCommon/VR.cpp`, `VR.h`**: The central hub for VR logic. Handles initialization and shutdown of VR SDKs (Oculus, OpenVR, VR920), manages HMD devices, and retrieves tracking data.
*   **`Source/Core/VideoCommon/VROculus.h`**: Header definitions for interacting with the Oculus SDK, including version-specific logic.
*   **`Source/Core/VideoCommon/VROpenVR.h`**: Header definitions for interacting with the OpenVR (SteamVR) SDK.
*   **`Source/Core/VideoCommon/VRGameMatrices.cpp`, `VRGameMatrices.h`**: Contains the complex matrix math for calculating the final view and projection matrices used in VR, transforming the game's camera into a VR-ready camera.
*   **`Source/Core/VideoBackends/D3D/VRD3D.cpp`, `VRD3D.h`**: Direct3D-specific implementation for VR. It handles creating D3D-compatible textures for the HMD, managing the texture swap chain, and submitting frames to the VR runtime.
*   **`Source/Core/VideoBackends/OGL/VROGL.cpp`, `VROGL.h`**: OpenGL-specific implementation for VR. The OpenGL equivalent of `VRD3D.cpp`.
*   **`Source/Core/VideoBackends/D3D/AvatarDrawer.cpp`, `AvatarDrawer.h`**: A new system for rendering 3D models (like VR controllers or hands) inside the game world using Direct3D.
*   **`Source/Core/VideoBackends/D3D/tiny_obj_loader.cpp`, `tiny_obj_loader.h`**: A third-party library for loading `.obj` 3D models, used by `AvatarDrawer`.
*   **`Source/Core/Core/ARBruteForcer.cpp`, `ARBruteForcer.h`**: Implements the Action Replay Brute Forcer tool.
*   **`Source/Core/Core/HideObjectEngine.cpp`, `HideObjectEngine.h`**: A new system to hide specific in-game objects based on their rendering commands, used to remove problematic UI elements in VR.
*   **`Source/Core/InputCommon/ControllerInterface/Vive/`, `Oculus/`, `Sixense/`, `OSVR/`**: New input drivers for various VR controllers.

#### Modified Direct3D Backend Files (`Source/Core/VideoBackends/D3D/`)

*   **`D3DBase.cpp` / `.h`**:
    *   Modified to directly support VR HMDs during device initialization.
    *   `Create()` function now has extensive logic to find the correct GPU and display associated with the HMD using its LUID and device name.
    *   Switched from `CreateDevice` and `CreateSwapChainForHwnd` to the lower-level `D3D11CreateDeviceAndSwapChain` to meet the requirements of VR runtimes.
    *   `Present()` logic was simplified, with stereo presentation now handled by the VR SDK via `VRD3D.cpp`.
*   **`D3DState.cpp`**:
    *   Added a `depth_clip_enable` flag to the rasterization state, allowing near-plane clipping to be disabled, which is critical in VR to prevent objects from disappearing when they get close to the player's face.
    *   Added a `m_layer_on_top` check to force certain objects (like a HUD) to render on top of everything else.
*   **`FramebufferManager.cpp` / `.h`**:
    *   Integrates VR initialization (`VR_StartFramebuffer`) and shutdown.
    *   Adds tracking for stereoscopic 3D (`m_stereo3d`, `m_eye_count`).
    *   Adds `m_frontBuffer[2]` to store eye textures for older VR SDKs.
*   **`Render.cpp` / `.h`**:
    *   `SwapImpl` was heavily modified to include a dedicated rendering path for VR.
    *   This path renders the scene once per eye, calling `VR_RenderToEyebuffer` to switch render targets.
    *   Implements the custom "Synchronous Timewarp" logic by repeatedly calling `VR_DrawTimewarpFrame` to re-project the last rendered frame.
    *   Adds logic to handle new stereo modes like `STEREO_OSVR`.
*   **`PixelShaderCache.cpp` / `.h`**: Added a new pixel shader (`s_OSVRProgram`) for OSVR-specific distortion.

#### Modified OpenGL Backend Files (`Source/Core/VideoBackends/OGL/`)

*The changes to the OpenGL backend largely mirror the Direct3D changes.*
*   **`FramebufferManager.cpp` / `.h`**: Same as D3D version; integrates VR init/shutdown and adds stereo rendering state.
*   **`Render.cpp` / `.h`**: `SwapImpl` is modified for VR, rendering once per eye into separate framebuffers and then submitting them. Includes the same custom timewarp logic as the D3D backend.
*   **`ProgramShaderCache.cpp`**: Modified to allow force-uploading of shader constants, needed for the VR camera updates.

#### Modified Common Video Files (`Source/Core/VideoCommon/`)

*   **`BPFunctions.cpp` / `.h`**:
    *   `ClearScreen` now has logic to handle the opcode replay feature, ensuring the screen is cleared correctly during replayed frames.
    *   A check for `bEFBCopyClearDisable` is added to prevent clearing the screen when certain EFB copy hacks are enabled.
*   **`GeometryShaderGen.cpp` / `.h`**:
    *   The geometry shader generator is now VR-aware. It can generate code for instanced stereo rendering (`gl_Layer` / `SV_RenderTargetArrayIndex`).
    *   It passes per-eye stereo parameters (`stereoparams`) to the shader for correct camera offsets.
*   **`VertexShaderManager.cpp` / `.h`**:
    *   This file sees some of the most significant changes. It is now responsible for the entire VR camera and world transformation pipeline.
    *   `SetConstants` is overhauled to calculate and apply the VR view matrix, which includes head tracking, world scaling, camera stabilization, and free-look.
    *   It now intelligently classifies draw calls to distinguish between the 3D world, skybox, and 2D HUD, applying different transformation matrices to each.
    *   It calculates and provides both the left and right eye projection matrices.
*   **`TextureCacheBase.cpp` / `.h`**:
    *   `CopyRenderTargetToTexture` now takes two rectangle arguments (`gameSrcRect` and `ourSrcRect`). This allows it to handle cases where VR mode has forced a full-screen render, even if the game only requested a small portion, ensuring the correct data is copied.
*   **`OpcodeDecoding.cpp` / `.h`**:
    *   The core opcode decoder is modified to support the "Opcode Replay" feature by adding awareness of `g_opcode_replay_frame` and logging commands to a buffer.
*   **`VideoConfig.cpp` / `.h`**:
    *   A massive number of new configuration options are added for VR, controlling everything from world scale and HUD position to motion sickness reduction techniques and controller visualization.
    *   Adds functions to save and reset game-specific VR settings.

#### Modified Core Files (`Source/Core/Core/`)

*   **`ActionReplay.cpp`**: Modified to apply the brute-forcer's generated patches.
*   **`ConfigManager.cpp` / `.h`**: Updated to load and save the new VR-specific settings.
*   **`Core.cpp` / `.h`**:
    *   `Init()` now calls `VR_Init()`.
    *   A separate `VRThread` is introduced to handle Asynchronous Timewarp on older SDKs.
    *   Adds a `KillDolphinAndRestart` function for the brute-forcer to recover from crashes.
*   **`Movie.cpp` / `.h`**: Adds a `SetFrameSkipping` function, as the VR implementation's timing control can conflict with traditional frame skipping.
*   **`PowerPC/*`**: Minor changes, mostly to the JIT and interpreter, to better handle idle skipping, which is often disabled in VR to achieve a more stable framerate for timewarping.

This comprehensive set of changes demonstrates a deep and thoughtful integration of VR into Dolphin's architecture, addressing not just the rendering challenges but also the crucial aspects of user comfort, performance, and game compatibility.
